Module migration.mapping#

This module provides the MigrationMap class and methods used to build and process a dict with data used as a sort of migration schema for data migration between odoo instances.

generate_full_map(model_name: str, target_model_name: str | None = None, recursion_level: int = 0) → dict[source]#

Make a fields map from a list of source and target fields. The output is intended to be feed to other migration methods / tools.

Parameters:

  • model_name (str) – The main source model name to map from.

  • target_model_name (str) – The main target model name to map to.

  • recursion_level (int) – The recursion level to apply. This means that the map will be made for the related models/fields too. - If 0, no recursion will be applied. - If 1, only the first level of related models/fields will be mapped. - If 2, the first and second level of related models/fields will be mapped. - …

Raises:

  • Exception – Raised when no executor instance is provided.

  • TooDeepException – Raised when the recursion level is not enough to traverse a relation field.

Returns:

A dict with the folling format: {modelname: {target_model, search_keys, fields, too_deep, removed, new}, …} where:

  • modelname is the source model name.

  • target_model is the target model name. You can modify it if you want to migrate to a different model name

  • search_keys is a dict with the keys to search for the records in the target model. Default is {‘id’: ‘id’, ‘name’: ‘name’}. This is useful to avoid data dups.

  • fields is a dict with the fields mapping: {‘source_field1’: ‘target_field1’, ‘source_field2’: ‘target_field2’, …}

  • too_deep is a list with the fields that are relations and the recursion level is not enough to traverse them. You can use this list to adjust the mappings.

  • removed is a list with the fields that are not in the target model. You can use this list to adjust the mappings.

  • new is a list with the fields that are not in the source model. You can use this list to adjust the mappings.

Return type:

dict

map = None#

Containes the mapping between the source and destination models/fields.

A map entry is a dict with the following structure:

{
    'source_model_name': {
        'search_keys': {'source_field1': 'destination_field1',
                        'source_field2': 'destination_field2', ...},
        'target_model': 'target_model_name',
        'fields': {
            'source_field': 'destination_field',
            ...
        },
        'removed': ['field1', 'field2', ...],
        'new': ['field1', 'field2', ...]
    }
}

normalice_fields(fields: list | dict) → dict[source]#

Normalice the given fields into the expected format. Which is (minimal conf):

{
    'fields': {'source_field_name1': 'target_field_name1', 'source_field_name2': transformer_function, ... }
}

Examples

in = [‘name’, {‘days’: ‘nb_days’}, ‘color’] out = {‘fields’: {‘name’: ‘name’, ‘days’: ‘nb_days’, ‘color’: ‘color’}}

in = [‘name’, {‘days’: ‘nb_days’}, {‘color’: color_transformer}] out = {‘fields’: {‘name’: ‘name’, ‘days’: ‘nb_days’, ‘color’: color_transformer}}

Parameters:

fields (Union[list, dict]) – The fields list to normalice.

Returns:

The normaliced fields dict structure.

Return type:

map (dict)
